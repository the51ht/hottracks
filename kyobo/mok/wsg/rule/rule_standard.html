표준규칙

표준규칙

기본 HTML 문서 정의

  • 기본문서 코드레이아웃 참고

DOCTYPE HTML5선언

  • HTML5로 선언한다.
  • DTD 선언 시 앞에 공백이나 다른 문자가 오지 않도록 해야하며, 문서 최상단에 선언한다.
								<!DOCTYPE html>

언어 정의

  • lang="ko"를 기본으로 표기하며 언어에 따라서 해당국가 언어로 표기한다.
  • https://www.w3schools.com/tags/ref_language_codes.asp (참고)
								<html lang="ko"> /* Korean 한국 */
								<html lang="en"> /* English 영어 */
								<html lang="ja"> /* Japanese 일본 */
								<html lang="zh"> /* Chinese 중국 */
								<html lang="ru"> /* Russian 러시아 */
								<html lang="de"> /* German 독일 */

인코딩 정의

  • charset="utf-8"를 기본 값으로 사용하고, <head>첫 번째에 선언한다.
								<meta charset="utf-8" />

브라우저 호환성 모드(PC)

								<meta http-equiv="X-UA-Compatible" content="IE=edge" />

뷰포트 정의

								<meta name="viewport" content="width=device-width,initial-scale=1.0,user-scalable=yes" />

문서정보 정의

  • <meta> : 최소한의 메타정보가 제공되어야 하며 상세내용은 기획과 협의필요.
  • keywords(키워드) : 페이지 키워드를 기재하고 콤마(,)로 구별하여 기술/동의어와 유의어를 포함.
  • description(페이지 정보 요약) : 구글검색에서 결과 페이지 내용 미리보기로 사용할 수도 있음.
  • 참고 : 구글 검색엔진 최적화 방법
								<meta name="Author" content="url" />
								<meta name="title" content="내용" />
								<meta name="keywords" content="내용" />
								<meta name="description" content="내용" />

표준 문법의 사용

  • HTML은 해당 DTD의 명세에 맞게 작성하며, W3C validation을 통과해야 한다.
  • DTD를 제외한 모든 엘리먼트와 애트리뷰트는 소문자로 작성한다.
  • 애트리뷰트의 값은 큰따옴표("")로 묶는다.
  • 특수기호는 Entity name을 사용하여 Entity 코드로 변환한다. Entity 코드는 ISO-8859-1을 기준으로 한다.
  • 마크업은 W3C html5문법에 준하여 사용한다.
  • 종료 태그는 생략하지 않는다.

구조, 표현, 동작의 분리

기본사항

  • HTML 구조와 디자인 표현, 동작의 구현은 분리하여 작업한다.
								<!-- 잘못된 예 -->
								<img width="200" height="100" />

								<button class="btn_something" onclick="something();">버튼</button>
								<script>
									function something() {

									}
								</script>

								<!-- 올바른 예 -->
								<img class="image" />
								.image {width: 200px; height: 100px;}

								<button class="btn_something">버튼</button>
								<script>
									$('.btn_something').on('click', function(event) {

									});
								</script>

예외사항

  • table의 col엘리먼트는 style속성으로 적용한다.
  • input[text], select 엘리먼트의 경우 너비의 고정폭 지정 허용한다.
  • 임시로 경고창을 띄워야 하는 경우, 사이즈를 지정해서 윈도우 팝업창을 띄울 경우는 예외로 한다.
								<table>
                                    <colgroup>
                                        <col style="width: 50%" />
                                        <col style="width: auto" />
                                    </colgroup>
								</table>
        
								<input type="text" id="formMyClass" class="my_class" style="width:80%;" />

								<!-- 예외사항 : 임시로 경고창을 띄워야 하는 경우 -->
								<a href="#" onclick="alert('준비중입니다.');return false;">링크</a>

								<!-- 사이즈를 지정해서 윈도우 팝업창을 띄울 경우 -->
								<a href="http://..." onclick="window.open(this.href,'pop','width=400,height=300');return false;" title="새 창으로 열기">팝업</a>

에디터 설정

  • 코드 들여쓰기 : 자식 요소는 1탭 들여쓰기 하며, 1탭의 크기는 공백 4칸으로 설정한다.
    • Webstorm : Editor > Code Style > Style Sheet[CSS, SCSS], HTML, JavaScript : Use tab character 체크, Tab size 값 "4"
    • MS VS Code : File > Preferences > Text Editor : Insert Spaces 체크 해제, Tab Size 값 "4"
  • 에디터의 자동 줄바꿈 라인은 120자로 한다.
    • Webstorm : Settings > Editor > Code Style "Hard wrap at" 값 "120"
    • MS VS Code : File > Preferences > Text Editor : Word Wrap Column 값 "120"
