HTML Markup 규칙

코드 규칙

코드 들여쓰기

  • 마크업의 중첩이 깊어질 때마다 자식 요소는 1탭 들여 쓰고, 1탭의 크기는 공백 4칸으로 설정한다.
  • 들여쓰기를 위한 공백문자는 Tab으로 통일하여 사용한다.
  • 모든 줄의 끝에는 공백문자(Space or Tab)를 포함하지 않는다.

들여쓰기를 하지 않는 경우

  • html의 자식 요소인 head, body
  • body의 자식 첫 번째 요소 <div id="wrapper">

속성의 개행

  • 속성의 개행은 에디터 설정의 자동 줄바꿈 라인을 기준으로 한다.
  • 단일 속성 단위로 개행하지 않는다.

주석 / 빈줄

  • 가독성을 고려하여 빈 줄을 추가 할 수 있다. 단, 1줄을 초과하지 않는다.
  • 블럭 구분을 위한 주석은 속성 개행 120자 라인에 맞춰 채운다. HTML 들여쓰기의 경우도
  • 블럭 구분을 위한 주석은 시작(s : )과 종료(e : )를 표시한다.
  • 코멘트가 있을경우 Start 주석 아래에 "DESC" 한줄 추가하여 주석처리

TODO 주석의 사용

  • TODO 주석을 기입할 경우 타인이 해당 내용을 이해할 수 있도록 상세하게 작성한다.
  • TODO 내용 해결 후 반드시 주석을 삭제한다.

수정/추가/삭제

  • 조치사항 진행 결과의 주석을 남긴다. (예 : [update or 수정][날짜(YYMMDD)][조치내용][이슈])
  • 잦은 수정사항을 고려하여 블럭단위 수정인 경우만 닫기주석으로 범위를 표시한다.

한줄 수정

블럭단위 수정

클래스나 단순 영역 수정

  • 수정 종료태그에는 변경 상세내용은 제외

단일항목 이슈단위 표시

  • 조치사항 이슈가 많은경우 이슈단위로 주석을 남긴다. (예 : [제목][조치내용][이슈])

수정 내용이 긴 경우

  • 조치사항 이슈가 많은경우 이슈단위로 주석을 남긴다. (예 : [제목][조치내용][이슈])
