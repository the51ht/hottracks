로컬 테스트 가이드

Node.js, Gulp 소개

Node.js

Node.js는 확장성 있는 네트워크 애플리케이션(특히 서버 사이드) 개발에 사용되는 소프트웨어 플랫폼이다. 작성 언어로 자바스크립트를 활용하며 논블로킹(Non-blocking) I/O와 단일 스레드 이벤트 루프를 통한 높은 처리 성능을 가지고 있다.

내장 HTTP 서버 라이브러리를 포함하고 있어 웹 서버에서 아파치 등의 별도의 소프트웨어 없이 동작하는 것이 가능하며 이를 통해 웹 서버의 동작에 있어 더 많은 통제를 가능케 한다.

이 프로젝트에선 내장된 패키지 매니저인 NPM(Node Package Manager)과 아래 소개하는 Gulp를 위해 사용한다.

Gulp

Gulp는 Node.js 기반 자동화 툴로써 각종 컴파일, minify, uglify 등 반복적으로 실행하는 작업을 자바스크립트 문법으로 테스크(Task) 등록하여 실행한다.

이 프로젝트에선 Sass SCSS 컴파일, CSS minify 및 map 파일 생성, 자바스크립트 uglify(.min.js 파일 생성) 그리고 BrowserSync를 통한 테스트 로컬 서버 실행을 담당한다.

프로그램 설치 / Gulp 명령어

프로그램 설치

Gulp 및 Gulp에서 사용되는 패키지 파일은 위에서 언급한 Nods.js의 NPM을 통해 설치하므로 Node.js를 우선 설치한다.

Node.js 설치

https://nodejs.org/ko/download/에서 장비 OS에 맞는 LTS(Long Term Support) 버전을 설치하거나 구축 시점 최신 LTS 버전인 Node.js 14.17.6을 장비 OS에 맞게 다운받아 설치한다.

일부 패키지는 Node 버전에 따라 실행이 안되는 경우가 있어 추후 최신 LTS 설치 시 설치된 패키지를 업데이트 해야할 가능성이 있으므로 Node.js 버전 통일을 권장한다.

NPM을 통한 패키지 설치

패키지를 설치하기 앞서 명령 프롬프트(cmd, Powershell 또는 terminal)를 실행하여 프로젝트 폴더 이동 후 npm init을 실행하여 package.json 파일을 생성해야 하나 Gulp 등 패키지 설치 정보가 설정된 package.json을 제공하므로 스킵한다.

프롬프트에서 Gulp 명령어를 실행하기 위해 npm install --global gulp-cli를 실행하여 장비 전역에서 Gulp 명령어를 실행 가능하게 설치한다.

프로젝트에 필요한 패키지를 설치해야하는 경우 해당 npm install `패키지명`을 입력하여 설치하게 되는데 퍼블리싱에선 HTML 구현에 필요한 패키지나 플러그인은 별도 관리하므로 개발에서만 사용할 수 있도록 npm install `패키지명` --save-dev 명령어를 통해 설치한다.

  • 아래 명령어는 gulpfile.js에 선언해놓은 Task를 통해 실행 가능한 명령어로 각각 다음과 같은 기능을 한다.
    • gulp : gulpfile.js 선언된 실행 가능한 Task 리스트 출력, 선택하여 해당 Task 실행한다. 내용은 아래와 같다.
    • gulp compile:scss : 프로젝트 폴더 내 *.scss 파일 *.css, *.min.css로 컴파일 처리한다
    • gulp compile:js : 프로젝트 폴더 내 *.js 파일을 *.min.js로 Uglify 처리한다.
    • gulp browse : browserSync 기능을 통한 로컬 테스트 서버를 실행한다. 서버가 실행된 상태에서 html, scss, js 파일 수정시 자동 리로드하여 브라우저에 적용한다.
      scss, js가 수정된 경우 compile:scss와 compile:js가 실행된 디 브라우저에 반영된다.
