네이밍 규칙

코드 네이밍 규칙

일반 규칙

  • 이름은 아래 표기법 정의에 따라 각 영문 대/소문자, 숫자, 언더스코어(_), 하이픈(-)을 사용하여 작성한다.
  • 이름은 시작은 영문 대문자, 숫자로 할 수 없다. (javascript 예외 사항은 제외)
  • 파스칼 표기법은 javascript의 클래스 함수 생성 외의 상황에서 사용하지 않는다. (javascript 네이밍 규칙 참고)

ID 네이밍 규칙

  • 카멜표기법(camelCase)을 사용하며 영문 대/소문자를 사용한다.
  • 개발 케이스에 따라서 중복 아이디가 사용될 수 있으므로 최상위 레이아웃, 폼 요소, anchor 등 필수 요소를 제외하고 사용을 최소화 한다.
  • 퍼블리싱에 생성한 ID는 기타 네이밍 규칙에 명시한다.

Class 네이밍 규칙

  • 체인 선택자(멀티클래스)의 개수는 3개까지 권장한다.
  • 클래스 예약어는 상황에 맞게 선택적으로 참고한다.
  • 상태를 나타내는 추가 클래스에 접두어 'is_'는 사용하지 않으며 활성화 관련 추가 클래스는 active로 통일한다.

CSS 네이밍 규칙

  • 하이픈 표기법(hyphen nate)을 사용하며 영문 소문자, 하이픈 및 숫자를 사용한다.
  • 영문 대문자 사용은 HTML id 셀렉터 사용 시만 가능하며 가급적 사용하지 않도록 한다.

javascript 네이밍 규칙

  • 카멜표기법(camelCase)을 기본으로 사용하며 영문 대/소문자, 언더스코어 및 숫자를 사용한다.
  • jQuery 객체 구분을 위해 $를 사용하지 않는다.
  • javascript의 클래스 함수의 경우 예외로 파스칼 표기법을 사용한다.
  • Private 변수의 경우 언더스코어로 시작한다. 함수와 함수 내 변수의 경우 Public/Private 구분 없이 언더스코어를 사용하지 않는다.
  • 블럭 변수(let)의 경우 for문에서의 사용을 IE11에서 지원하지 않으므로 사용에 유의한다.

파일 및 폴더 네이밍 규칙

  • 스네이크 표기법(underscore)을 사용하며 영문 소문자, 언더 스코어 및 숫자를 사용한다.
  • Sass 파일(.scss) 중 다른 파일에 @import 되는 분할 파일(Partial File)의 파일명은 언더스코어(_)로 시작한다.
    • eg. _buttons.scss, _header.scss
  • 이미지 파일의 경우 파일명의 규칙은 기능_의미_형태_번호_상태.확장자순으로 하며 형태, 번호, 상태는 생략할 수 있다.
    • eg. text_join_process_01.jpg, btn_cancel_black_off.gif, tab_bullet_circle_active.gif, sort_arrow_up.png
