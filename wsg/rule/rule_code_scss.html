SCSS 코드 규칙

개요

본 가이드는 Sass 문법 표준 규칙을 안내하고 프로젝트 진행을 원활하게 하기 위해 Sass 문법의 일부를 설명하지만 모든 Sass 문법에 대해 설명하지 않습니다.

프로젝트는 CSS 문법과 유사한 SCSS 문법으로 작업을 진행합니다.

아래 예시중 "아웃풋 CSS"의 내용은 가독성을 위해 CSS 표준규칙을 무시하고 개행 처리되었습니다.

코드 규칙

코드 들여쓰기

  • 1탭 들여쓰기 하며, 1탭의 크기는 공백 4칸으로 설정한다.

줄바꿈

  • 모든 속성은 줄바꿈 처리한다.
  • 박스단위 구분을 위해 단 사이 빈줄 추가(최대 2줄을 넘지않는다)
  • Nesting 구조에서 속성과 클래스 사이, 클래스와 클래스 사이는 반드시 한줄 빈칸을 추가한다.

주석

주석

  • 한줄 주석과 두줄 이상의 주석은 아래와 같이 표기한다.
  • Sass의 경우 CSS에서 지원하지 않는 // 주석 사용이 가능하며 해당 주석은 CSS로 컴파일 시 포함되지 않는다.
  • DESC, TODO 등 참고용 주석은 반드시 // 주석으로 작성한다.
  • /* */ 주석은 컴파일시 주석을 작성한 위치에 표시되므로 블럭 최상단 클래스에만 작성한다.

블럭 구분

  • 코드 가독성을 위해 블럭을 구분하는 경우 아래와 같이 표시한다.
  • 주석 시작과 끝(/*, */)라인의 *는 에디터 설정의 자동 줄바꿈 라인에 맞춰 채운다.
  • 블럭 구분 주석의 경우 최상위 클래스에만 작성한다

TODO 주석의 사용

  • TODO 주석을 기입할 경우 타인이 해당 내용을 이해할 수 있도록 상세하게 작성한다.
  • TODO 내용 해결 후 반드시 주석을 삭제한다.

상세 코드 규칙

작은 따옴표('') 사용 범위

폰트의 선언, filter, content에 작은 따옴표('')를 사용한다.

background 속성

background 속성에는 기본 따옴표를 사용하지 않으며 공백과 괄호 등의 이슈가 발생하는 케이스에 대하여 작은 따옴표를 허용한다.

세미콜론(;)

마지막 선언된 속성에도 세미콜론(;)을 사용한다.

공백의 사용

선택자 간, 중괄호간 한칸의 공백을 사용한다, 중괄호 안쪽 좌우에는 공백은 사용하지 않는다.

선택자

공통 선택자

'*' 공통 선택자는 웹 페이지의 성능을 저하를 가져오기 때문에 가급적 사용하지 않는다.

At(@) 규칙

@import

Sass 파일은 모듈별로 파일을 분리하여 제작하고 이를 최상위 Sass 파일에서 불러들여 사용하며 @import로 파일을 불러들여 컴파일 한다.

  • 현재 Sass 팀은 @import를 아래와 같은 이유로 권장하지 않으며 아래 소개할 @use를 사용하도록 권장하고 있다.
    • @import는 모든 변수, 믹스인 및 함수를 전역적으로 액세스할 수 있습니다. 이로 인해 변수, 믹스인, 함수에 대해 정의된 위치를 말하기가 매우 어렵습니다.
    • 모든 것이 전역적이기 때문에 라이브러리에서 충돌을 피하기 위해 모든 멤버에 접두어를 붙여야 합니다.
    • @extend 규칙도 전역적이므로 확장될 스타일 규칙을 예측하기 어렵습니다.
    • 각 스타일시트는 @import할 때마다 실행되고 CSS가 내보내지므로 컴파일 시간이 늘어나고 출력이 부풀려집니다.
    • 다운스트림 스타일시트에 액세스할 수 없는 private 멤버 또는 placeholder를 정의할 방법이 없었습니다.

@import 사용을 권장하지는 않지만 문법상 @use와 크게 다르지 않다.

@mixin과 @include

@mixin을 사용하여 전체 스타일시트에서 재사용할 수 있는 스타일을 정의할 수 있다.

@mixin <명칭> { ... } 또는 @mixin <명칭> (인자값...) { ... }으로 작성할 수 있다.
