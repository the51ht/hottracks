태그 소개

참고 URL

  • 검색엔진 최적화(SEO) 기본 가이드
  • 구글에서 인식하는 모든 메타 태그
  • 구글 모바일 친화성 테스트
  • 페이스북 웹 마스터용 공유 가이드
  • 페이스북 공유 디버거

기본 태그

아래의 태그는 HTML 문서의 <head> 요소 내에 배치한다.

<title> 태그

  • <title> 태그는 사용자와 검색엔진에 모두 특정 페이지의 주제가 무엇인지 알려준다.
  • 사이트의 각 페이지에 사용할 고유한 제목을 만들어야 한다.
  • 문서가 검색결과 페이지에 표시될 때 검색결과의 첫 번째 줄에 title 태그의 내용이 표시될 수 있다.

상황별 title 예시

								<title>SEO title 기획 정의 후 추가</title>

<meta name="description" /> 태그

  • 페이지의 description 메타 태그는 Google 및 다른 검색엔진에 페이지 내용을 요약하여 제공한다.
  • 페이지 제목은 단어 몇 개나 문구로 이뤄질 수 있지만 페이지의 description 메타 태그는 한두 문장 또는 짧은 단락 정도로 길어질 수 있다.

상황별 description 예시

								<meta name="description" content="SEO description 기획 정의 후 추가" />

<meta name="subject" /> 태그

  • 이 태그는 웹사이트의 주제를 선언하는 데 사용한다.

상황별 subject 예시

								<meta name="subject" content="SEO subject 기획 정의 후 추가" />

<meta name="keywords" /> 태그

  • 화면 주제 단어, 문장을 입력한다.

<meta name="author" /> 태그

  • 페이지 작성자의 이름을 지정하는 데 사용한다.

상황별 author 예시

								<meta name="author" content="SEO author 기획 정의 후 추가" />

<meta name="rating" content="adult" /> 태그

  • 성인컨텐츠인 경우 추가되어야 하는 메타태그

전체 태그 예시
  • 기존에 사용되던 <meta name="keywords">의 경우 트래픽 유입을 위해 무리하게 키워드를 넣는 악용사례가 많아 현재 검색엔진에서 사용하지 않는 메타태그로 정의하지 않음.

Open Graph protocol Metadata 태그

참고 링크 : The Open Graph protocol

<meta name="og:title" />

  • 기본 태그 <title>과 동일

<meta name="og:type" />

  • 개체의 유형을 표시하는 메타 태그
  • music.album, video.movie, book, profile, website 등의 값을 가질 수 있다.
  • 자세한 사항은 https://ogp.me/#types 참고

<meta name="og:url" />

  • 페이지의 영구적인 URL 값

<meta name="og:description" />

  • 기본 태그 <meta name="description">과 동일

<meta name="og:locale" />

  • 사이트 사용자 언어 태그. 기본값은 en_US, 한글의 경우는 ko_KR로 설정.
  • 참고링크 : Territory-Language Information

<meta name="og:locale:alternate" />

  • 사이트를 사용할 수 있는 다른 언어 태그 배열로 중복 선언 가능.

<meta name="og:site_name" />

  • 화면이 더 큰 웹 사이트의 일부인 경우 전체 사이트에 표시되어야 하는 이름

<meta name="og:image" />, <meta name="og:image:url" />

  • 화면을 나타내는 이미지의 URL

<meta name="og:image:secure_url" />

  • 화면에서 HTTPS가 필요한 경우 사용할 대체 URL

<meta name="og:image:width" />

  • 이미지의 넓이 값(숫자)

<meta name="og:image:height" />

  • 이미지의 높이 값(숫자)

전체 태그 예시

								<meta property="og:title" content="The Rock" />
								<meta property="og:type" content="video.movie" />
								<meta property="og:url" content="https://www.imdb.com/title/tt0117500/" />
								<meta property="og:description" content="The Rock: Directed by Michael Bay. With Sean Connery, Nicolas Cage, Ed Harris, John Spencer. 								<meta property="og:locale" content="en_US" />
								<meta property="og:locale:alternate" content="ko_KR" />
								<meta property="og:site_name" content="IMDb" />

Twitter Card Metadata 태그

참고 링크 : About Twitter Cards, Twitter 카드 테스트

<meta name="twitter:card" /> 필수

  • 트위터 카드 노출 유형 선택. “summary”(기본값), “summary_large_image”, “app”, “player”

<meta name="twitter:title" /> 필수

  • 기본 태그 <title>과 동일

<meta name="twitter:site" />

  • 사이트와 관련한 트위터 아이디 (@Twitter)

<meta name="twitter:description" />

  • 트윗 내에서 프레젠테이션에 적합하도록 콘텐츠를 간결하게 요약한 설명으로 제목을 다시 사용하거나 웹사이트에서 제공하는 일반 서비스를 설명하는 데 사용해서는 안 된다.
  • 플랫폼 별 동작
    • iOS, Android: 표시되지 않음
    • 웹: 타임라인 및 확장된 트윗에서 두줄 이상 말줄임(...)

<meta name="twitter:image" />

  • 가로 세로 비율이 1:1인 이미지 URL로 최소 사이즈 144x144에서 최대 사이즈 4096x4096이며 최대 용량 5MB 이하여야 한다.
  • JPG, PNG, WEBP, GIF 포멧을 지원하며 SVG는 지원하지 않는다. 움직이는 GIF의 경우 첫 프레임이 노출된다.

<meta name="twitter:image:alt" />

  • 이미지 대체 텍스트로 최대 420자까지 지원한다.
